第 7 章 适配器与外观模式

  • 一、基础
  • 二、实践
    • 2.1 适配器需求
    • 2.2 适配器代码
    • 2.3 外观需求
    • 2.4 外观代码

适配器：封装对象，并提供不同的接口
外观：简化一群类的接口

一、基础

  • 适配器模式
将一个类的接口，
转换成客户期望另一个接口。
适配器让原本不兼容的类可以协作无间。
  • 外观模式
提供了一个统一的接口
用来访问子系统的一群接口。
外观定义了一个高层接口，
让子系统更容易使用。

二、实践

2.1 适配器需求

使用系统弹窗，提示消息

迭代
使用漂亮的第三方弹窗系统，提示消息

2.2 适配器代码

  • 结合实际，举个简单的例子
@interface CZHUD : NSObject

- (void)showAlterWithMessage:(NSString *)message;

@end


@implementation CZHUD

- (void)showAlterWithMessage:(NSString *)message {
    NSLog(@"弹窗消息：%@", message);
    // 封装了一个 适配器，后面改用不同的弹窗都不影响原代码
}

@end

2.3 外观需求

家庭影院
有投屏、音响、电影播放器

迭代
换了一套音响

2.4 外观代码

@implementation MovieAppearance

// 实际对外使用，封装好了 播放电影的各种提前处理，比如开机，对焦等
- (void)playMovie {
    MovieScreen *screen = [[MovieScreen alloc] init];
    [screen open];
    [screen focusing];
    [screen brightness];
    
    MovieSound *sound = [[MovieSound alloc] init];
    [sound open];
    [sound volume];
    
    MoviePlayer *player = [[MoviePlayer alloc] init];
    [player play];
}

@end
